# Estratégia de copy — TZOLKIN como marca de software

Série de 6 documentos que reposiciona a copy do site: de "agência de landing pages/tráfego" para **marca de software de alto padrão** com 4 frentes — Serviços (consultoria/assessoria), Produtos (white-label/personalizados), Ferramentas próprias (SaaS/PWAs/NaaS — em breve) e Educacional (em breve).

## Documentos

1. [01-arquitetura-de-marca.md](01-arquitetura-de-marca.md) — posicionamento, message house, descriptor recomendado ("TZOLKIN — Software de alto padrão"), regra anti-vaporware, elevator pitch.
2. [02-hero-e-narrativa-home.md](02-hero-e-narrativa-home.md) — 3 opções de hero (recomendado: "Software de alto padrão."), fluxo novo da home seção a seção, headlines verbatim, o que sai da home.
3. [03-servicos-consultoria.md](03-servicos-consultoria.md) — seção de serviços reescrita: consultoria, assessoria contínua, desenvolvimento sob medida + faixa "Áreas de atuação" (IA → cybersecurity); destino do /catalogo.
4. [04-produtos-e-ferramentas.md](04-produtos-e-ferramentas.md) — framing Produtos vs. Ferramentas; seção nova "Ferramentas TZOLKIN" com 4 cards "Em breve" + lista de espera; pricing reposicionado como produtos de software.
5. [05-navegacao-seo.md](05-navegacao-seo.md) — menu novo (4 frentes + Cases + Institucional + CTA "Iniciar projeto"), footer, metadata verbatim da home e das páginas /ferramentas e /educacional, mapa de CTAs, URLs.
6. [06-guia-de-voz.md](06-guia-de-voz.md) — voz em 3 adjetivos (direta, precisa, sofisticada), política PT/EN, regras mecânicas, tabela "não diga/diga" com 18 exemplos reais reescritos, checklist de publicação.

## Decisões centrais (comuns a todos os docs)

- **A palavra que faltava: "software".** Ela entra no hero, na meta description e no descriptor — é ela que desfaz a leitura de "agência".
- **Tráfego/CRO não sai do site** — deixa de ser a identidade e vira demonstração de diagnóstico dentro da frente Serviços ("Tráfego não é solução, é amplificador" permanece).
- **"Em breve" sem vaporware:** ferramentas e educacional têm presença de identidade (cards + lista de espera), mas zero data, zero feature, zero pré-venda.
- **Transparência de preços vira prova do posicionamento** — escopo fechado é comportamento de produto de software, não de agência.
- **Voz:** PT-BR padrão com termos técnicos EN consolidados permitidos; sentence case; direta, precisa, sofisticada.
